## Design Context

### Users
Developers flashing MediaTek devices — technical users who understand chipset, storage, slot, flash/wipe actions. They use this tool in a focused, task-driven context: load a scatter file, review partitions, confirm the plan summary, and flash. The footer is their final checkpoint before committing.

### Brand Personality
Modern & Confident — polished, clean, premium dev-tool aesthetic. Professional tooling feel like Sentry, Linear, or VS Code's status bar. No-nonsense but refined.

### Aesthetic Direction
- **Tone**: Minimal, precise, utilitarian with premium polish
- **Theme**: Dark mode primary (developers in dimly lit rooms), light mode secondary
- **Anti-references**: Heavy card shadows, gradient accents, decorative elements, over-designed status indicators

### Design Principles
1. **Information density with breathing room**: Technical UIs need to pack info but not feel cramped. Use generous spacing between groups, tight spacing within.
2. **The action is the hero**: The Flash CTA should always be the most visually prominent element in the footer — color, shadow, and weight should draw the eye there.
3. **Spacing creates hierarchy**: Use a clear spacing rhythm (tight groupings, generous separations) to communicate which items are related.
4. **Consistency over cleverness**: A single responsive grid pattern, consistent card heights, and a semantic spacing scale. No one-off hacks.
5. **Subtle polish**: The interface should feel intentional through spacing, not decoration. No side borders, no gradient text, no glassmorphism.
